# Tomilo Lib Server

Tomilo Lib Server - это backend для онлайн-библиотеки манги, комиксов и других форматов цифровых публикаций. Проект предоставляет полнофункциональную платформу с системой пользователей, закладками, историей чтения и автоматическим парсингом контента.

## 🛠 Технологии

- **Backend**: NestJS, TypeScript
- **Database**: MongoDB с Mongoose
- **Authentication**: JWT, OAuth (Yandex, VK), bcrypt
- **File Upload**: Multer
- **Validation**: class-validator, class-transformer
- **Real-time**: WebSocket (Socket.IO)
- **Scheduling**: @nestjs/schedule
- **Logging**: Winston

## 🚀 Основные функции

### 👥 Пользователи
- Регистрация и авторизация (JWT, OAuth через Yandex и VK)
- Профиль с аватаром, закладками и историей чтения
- Система ролей (пользователь, администратор)
- Статистика чтения и пользовательская активность

### 📚 Тайтлы (Манга/Комиксы)
- CRUD операции для тайтлов
- Поиск и фильтрация по жанрам, статусу, типу
- Пагинация и сортировка
- Рейтинги, просмотры и комментарии
- Возрастные ограничения
- Обложки и описания

### 📖 Главы
- Создание глав с загрузкой изображений
- Навигация между главами (следующая/предыдущая)
- Статистика просмотров
- Автоматическое обновление счетчика глав в тайтлах
- Массовое удаление глав

### 🔖 Закладки и история
- Добавление/удаление закладок
- История чтения с отслеживанием прогресса
- Статистика пользователя
- Очистка истории чтения

### 🔍 Поиск и коллекции
- Поиск тайтлов по названию и описанию
- Коллекции тайтлов с обложками
- Комментарии к коллекциям

### 🔄 Автоматический парсинг
- Импорт тайтлов и глав с внешних источников
- Автоматическая проверка обновлений
- Поддержка различных сайтов-источников

### 💬 Комментарии
- Комментарии к тайтлам и главам
- Лайки/дизлайки комментариев
- Вложенные комментарии (ответы)

### 🔔 Уведомления
- Система уведомлений для пользователей
- Отслеживание непрочитанных уведомлений
- Массовая отметка уведомлений как прочитанных

## 📊 API Endpoints

Подробное описание всех API endpoints доступно в файле [API_ENDPOINTS.md](API_ENDPOINTS.md).

### Аутентификация
- `POST /auth/register` - Регистрация
- `POST /auth/login` - Вход
- `POST /auth/yandex` - OAuth через Yandex
- `POST /auth/vk` - OAuth через VK

### Тайтлы
- `GET /titles` - Список тайтлов с пагинацией
- `POST /titles` - Создание тайтла (admin)
- `GET /titles/:id` - Получить тайтл
- `PUT /titles/:id` - Обновить тайтл (admin)

### Главы
- `POST /chapters/upload` - Создание главы с файлами (admin)
- `GET /chapters/title/:titleId` - Главы тайтла
- `POST /chapters/:id/view` - Увеличить просмотры

### Пользователи
- `GET /users/profile` - Профиль текущего пользователя
- `GET /users/profile/bookmarks` - Закладки пользователя
- `POST /users/profile/bookmarks/:titleId` - Добавить закладку

## 🏃‍♂️ Быстрый старт

1. **Установка зависимостей**
   ```bash
   npm install
   ```

2. **Запуск MongoDB**
   ```bash
   docker-compose up -d
   ```

3. **Настройка окружения**
   Создайте файл `.env` на основе `.env.example`:
   ```
   MONGO_LOGIN=admin
   MONGO_PASSWORD=password123
   MONGO_HOST=localhost
   MONGO_PORT=27017
   MONGO_AUTHDATABASE=admin
   JWT_SECRET=your-secret-key
   ```

4. **Запуск приложения**
   ```bash
   npm run start:dev
   ```

## 💾 Бэкапы и миграции

Перед любыми миграциями БД нужно сделать бэкап.

**Разовый бэкап**
```bash
npm run backup:db
```
Дамп сохраняется в каталог `backups/dump-YYYY-MM-DD/<database>/` (или в `BACKUP_DIR` из `.env`). Используются только Node.js и Mongoose, внешние утилиты не нужны.

**Миграция с предварительным бэкапом**
```bash
npm run migrate:reading-history:safe   # сначала бэкап, затем миграция истории чтения
```

**Ежедневный бэкап (cron)**  
Установите права и добавьте в crontab (`crontab -e`), например на 02:00:
```bash
chmod +x scripts/cron-daily-backup.sh
# в crontab:
0 2 * * * /path/to/tomilo-lib-server/scripts/cron-daily-backup.sh >> /var/log/tomilo-backup.log 2>&1
```

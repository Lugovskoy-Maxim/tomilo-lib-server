# Интеграция бекапов БД с S3

## Обзор

Система бекапов была модернизирована для загрузки архивов в облачное хранилище S3 с отправкой ссылки по почте вместо вложения файла. Это решает проблему ограничения размера вложений в email.

## Архитектура

1. **S3BackupService** - новый сервис для создания бекапов и загрузки в S3
2. **Обновленный DatabaseBackupService** - поддерживает два режима:
   - Старый режим: отправка вложения по почте
   - Новый режим S3: загрузка в облако + отправка ссылки
3. **Новый метод в EmailService** - `sendDatabaseBackupS3Link()` для отправки email со ссылкой

## Настройка

### 1. Переменные окружения

Добавьте в `.env` следующие переменные:

```bash
# S3 Backup Configuration
BACKUP_USE_S3=true
BACKUP_S3_PREFIX=backups/database
BACKUP_S3_CLEANUP_ENABLED=true
BACKUP_S3_DAYS_TO_KEEP=30

# Существующие переменные S3 (должны быть уже настроены)
S3_ENDPOINT=https://s3.regcloud.ru
S3_ACCESS_KEY_ID=your_access_key_id
S3_SECRET_ACCESS_KEY=your_secret_access_key
S3_BUCKET=your_bucket_name
S3_PUBLIC_URL=https://your_bucket_name.s3.regcloud.ru
```

### 2. Включение бекапов

Убедитесь, что бекапы включены:
```bash
BACKUP_EMAIL_ENABLED=true
BACKUP_EMAIL_TO=support@tomilo-lib.ru
```

## Режимы работы

### Режим S3 (рекомендуется)

При `BACKUP_USE_S3=true`:
1. Создается бекап БД
2. Архивируется в `.tar.gz`
3. Загружается в S3 по пути: `{BACKUP_S3_PREFIX}/{YYYY-MM-DD}/tomilo-db-{timestamp}.tar.gz`
4. На email отправляется ссылка для скачивания
5. Старые бекапы автоматически удаляются (если `BACKUP_S3_CLEANUP_ENABLED=true`)

### Старый режим (вложения)

При `BACKUP_USE_S3=false` или если S3 не настроен:
1. Создается бекап БД
2. Архивируется в `.tar.gz`
3. Отправляется как вложение в email
4. Если размер превышает `BACKUP_EMAIL_MAX_BYTES`, отправляется уведомление без вложения

## Использование

### Автоматические бекапы (cron)

Ежедневный бекап запускается в 03:00 через `DatabaseBackupScheduler`. Режим определяется переменной `BACKUP_USE_S3`.

### Ручной запуск через скрипт

```bash
# Создать бекап (только локально)
npm run backup:db

# Создать бекап и загрузить в S3
npx ts-node -r tsconfig-paths/register scripts/backup-db.ts --s3

# Создать бекап, загрузить в S3 и очистить старые
npx ts-node -r tsconfig-paths/register scripts/backup-db.ts --s3 --cleanup --days 30
```

### Программное использование

```typescript
import { S3BackupService } from './backup/s3-backup.service';

// Создать и загрузить бекап
const result = await s3BackupService.createAndUploadBackup();
console.log(`Бекап загружен: ${result.url}`);

// Очистить старые бекапы
await s3BackupService.cleanupOldBackups(30);
```

## Миграция со старой системы

1. Установите переменную `BACKUP_USE_S3=true`
2. Убедитесь, что S3 настроен корректно
3. Существующий код продолжит работать, автоматически переключившись на S3
4. Старые бекапы на почте больше не будут превышать лимиты размера

## Безопасность

- Файлы в S3 загружаются с ACL `public-read` (ссылки публичные)
- Рекомендуется использовать S3 с приватным доступом и пре-подписанными URL при необходимости
- Ключи S3 должны храниться в секретах окружения

## Мониторинг

- Логируются все этапы создания и загрузки бекапа
- При ошибках отправляется email с деталями
- Размер бекапа указывается в email-уведомлении

## ⚡ Новая фича: FILE_STORAGE_ONLY_S3 (режим "только S3" для пользовательских файлов)

### Настройка режима хранения файлов

Добавлена переменная `FILE_STORAGE_ONLY_S3` для полного перехода на облачное хранение:

```
FILE_STORAGE_ONLY_S3=false  # Локально + дубликат в S3 (текущая логика)
FILE_STORAGE_ONLY_S3=true   # ТОЛЬКО S3, локальная папка /uploads/ не используется
```

**Что меняется при `FILE_STORAGE_ONLY_S3=true`:**
- Файлы сохраняются **только в S3** (аватары, обложки, страницы глав)
- Локальная папка `/uploads/` остается пустой
- Все методы возвращают **S3 public URL** вместо локальных путей
- Cron-синхронизация отключена (не нужна)
- Delete операции только в S3

**Преимущества:**
- 0 дискового пространства на сервере
- Автоматическое масштабирование
- CDN-ready (публичные ссылки)

**Использование:**
1. Добавьте в `.env`: `FILE_STORAGE_ONLY_S3=true`
2. Перезапустите сервер: `npm run start:dev`
3. Загружайте файлы – все пойдет в S3

**getUserAvatarPath() и подобные** автоматически вернут S3 URL.

## Ограничения (бэкапы)

- Требуется настроенный S3-клиент (уже есть в проекте)
- Размер бекапа ограничен только квотой S3
- Ссылки в email могут попадать в спам (проверьте настройки почтового сервера)
